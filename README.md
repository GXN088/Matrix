# Matrix
В этой задаче тебе нужно:

Реализовать интерфейс DBObject в классе User.
Реализовать метод initializeIdAndName так, чтобы программа работала и выводила на экран: "The user's name is Neo, id = 1".
Метод toString и метод main менять нельзя.
Подумать, что должен возвращать метод initializeIdAndName в классе User.
Обрати внимание: методы toString() и main() менять нельзя.

Псевдокод:

```
КЛАСС Solution:
    МЕТОД main(args):
        ВЫВЕСТИ Matrix.NEO
        КОНЕЦ МЕТОДА
    КОНЕЦ КЛАССА

    СТАТИЧЕСКИЙ КЛАСС Matrix:
        ПУБЛИЧНОЕ СТАТИЧЕСКОЕ ПОЛЕ NEO = СОЗДАТЬ User().initializeIdAndName(1, "Neo")
    КОНЕЦ КЛАССА

    ИНТЕРФЕЙС DBObject:
        МЕТОД initializeIdAndName(id, name) ВОЗВРАЩАЕТ DBObject
    КОНЕЦ ИНТЕРФЕЙСА

    СТАТИЧЕСКИЙ КЛАСС User РЕАЛИЗУЕТ DBObject:
        ПОЛЕ id (целое число)
        ПОЛЕ name (строка)

        МЕТОД initializeIdAndName(id, name):
            УСТАНОВИТЬ this.id = id
            УСТАНОВИТЬ this.name = name
            ВЕРНУТЬ this
        КОНЕЦ МЕТОДА

        МЕТОД toString():
            ВЕРНУТЬ "The user's name is " + name + ", id = " + id
        КОНЕЦ МЕТОДА
    КОНЕЦ КЛАССА
КОНЕЦ КЛАССА Solution
```

Пошаговое описание логики:

1. Запуск программы - вызывается метод main()
2. Обращение к Matrix.NEO - получаем статическое поле из класса Matrix
3. Создание объекта User - при обращении к Matrix.NEO:
   · Создается новый объект класса User
   · Немедленно вызывается его метод initializeIdAndName(1, "Neo")
4. Инициализация User:
   · Устанавливается id = 1
   · Устанавливается name = "Neo"
   · Метод возвращает ссылку на этот же объект (this)
5. Вывод результата - при выводе Matrix.NEO вызывается метод toString(), который форматирует строку с именем и ID

Ключевые моменты:

· Используется паттерн Fluent Interface (метод возвращает this)
· Реализация интерфейса DBObject в классе User
· Ленивая инициализация не используется - объект создается сразу при загрузке класса Matrix
